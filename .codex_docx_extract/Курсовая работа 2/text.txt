Квалификационные характеристики проекта
Введение
Актуальность проекта:
В современном образовательном процессе студенты и преподаватели ежедневно сталкиваются с большим потоком информации о различных мероприятиях: лекциях, практических занятиях, экзаменах и консультациях. Традиционные способы передачи этой информации (бумажные расписания, стенды, сообщения в мессенджерах) часто неудобны, теряются или не позволяют оперативно вносить изменения.
Разработанное мобильное приложение «Календарь учебных мероприятий колледжа» призвано решить эту проблему. Оно позволяет хранить мероприятие и расписание в цифровом виде, добавлять события, фильтровать их по типу (Лекции, Практика, Экзамены) и привязывать к конкретной дате. Поддержка светлой и тёмной темы делает приложение удобным для использования в любых условиях освещенности.
Таким образом, создание данного приложения является актуальным, так как оно способствует цифровизации образовательной среды колледжа ИРКПО, упрощает планирование времени студентов и повышает их успеваемость за счет лучшей организованности.
Цель проекта:
Разработать мобильное приложение для Android для ведения календаря учебных мероприятий с возможностью управления событиями и фильтрации по дате и типу.
Задачи проекта:
Провести анализ предметной области и определить сущность «Учебное мероприятие».
Разработать визуальный интерфейс (календарь, форма добавления, карточки событий).
Реализовать логику добавления, редактирования и удаления событий.
Реализовать функцию фильтрации событий по выбранной дате и по типу.
Реализовать поддержку светлой и тёмной темы оформления.
Провести тестирование приложения на эмуляторе Android.
Анализ предметной области
Описание проблемы:
В образовательном процессе студентам и преподавателям требуется оперативно отслеживать большое количество учебных мероприятий. Существующие способы имеют недостатки:
Неактуальность данных: о событии могут узнать уже после его проведения.
Отсутствие персонализации: стандартные календари не позволяют выделять учебные события отдельным типом или цветом.
Неудобство управления: нет возможности добавить к событию личный комментарий, аудиторию или преподавателя.

Спецификация требований к программному продукту
Функциональные требования:
Управление событиями: пользователь должен иметь возможность создавать новые события (название, дисциплина, дата, время, аудитория, преподаватель, тип, описание), редактировать их и удалять.
Отображение календаря: приложение должно отображать календарь для выбора даты.
Фильтрация списка по дате: при выборе даты в календаре список событий должен обновляться, показывая только события за этот день.
Фильтрация по типу: пользователь должен иметь возможность фильтровать события по типу (Лекция, Практика, Экзамен) с помощью чипов.
Смена темы: в приложении должна быть возможность переключения между светлой и тёмной темами интерфейса.
Просмотр деталей: при нажатии на событие открывается экран с подробной информацией.
Нефункциональные требования
Производительность: Загрузка календаря и списка событий должна происходить мгновенно (менее 1 секунды).
Надёжность: Приложение не должно вылетать при вводе некорректных данных (например, пустое название). Должна быть реализована валидация полей.
Юзабилити: Интерфейс должен быть интуитивно понятен студенту первого курса.
Совместимость: Приложение работает на устройствах с Android версии 8.0 (API 26) и выше.
Требования к интерфейсам
Пользовательский интерфейс должен соответствовать гайдлайнам Material Design.
Использовать карточки (CardView) для отображения событий.
Предусмотреть плавающую кнопку добавления (FAB) для быстрого перехода к созданию события.
Навигация между экранами организована логично и последовательно.

Проектирование программного модуля
Архитектура приложения:
Для хранения данных о мероприятиях была спроектирована модель Event на языке Java.
Основные поля модели Event:
id (int): Уникальный идентификатор события.
title (String): Название мероприятия.
discipline (String): Название дисциплины (предмета).
type (EventType): Тип мероприятия (Лекция/Практика/Экзамен). Реализован через перечисление enum.
teacher (String): Преподаватель.
audience (String): Аудитория.
date (String): Дата проведения.
time (String): Время проведения.
description (String): Подробное описание.

private void filterEventsByType(EventType type) {
filteredEvents.clear();
for (Event event : allEvents) {
if (event.getDate().equals(selectedDate) && event.getType() == type) {
filteredEvents.add(event);
}
}
adapter.setEvents(filteredEvents);

Руководство пользователя
Системные требования:
Операционная система: Android 8.0 (API 26) или выше.
Оперативная память: не менее 1 ГБ.
Свободное место: не менее 20 МБ.
Процесс установки:
Скопировать APK-файл на устройство.
Разрешить установку из неизвестных источников (Настройки → Безопасность → Неизвестные источники).
Открыть APK-файл и следовать инструкциям.
Примеры использования:
Добавление события: Нажать на плавающую кнопку «+» (FAB) на главном экране. Заполнить поля: название, дисциплина, дата, время, аудитория, преподаватель, тип, описание. Нажать «Сохранить».
Просмотр событий на дату: Выбрать любую дату в календаре. События, назначенные на этот день, отобразятся в списке внизу.
Фильтрация по типу: Нажать на чип «Лекция», «Практика» или «Экзамен» над списком.
Просмотр деталей: Нажать на событие в списке. Откроется экран с подробной информацией.
Редактирование: На экране деталей нажать кнопку «Редактировать».
Удаление: На экране деталей нажать кнопку «Удалить» и подтвердить действие.
Смена темы: Нажать на три точки в правом верхнем углу (меню) и выбрать «Темная тема» или «Светлая тема».
