Техническое задание на курсовую работу
Тема: «Электронное приложение «Календарь событий колледжа»
1. Введение
Разработка электронного приложения «Календарь событий колледжа» предназначена для организации и отслеживания учебных и внеучебных мероприятий студентами и преподавателями. Приложение позволяет фиксировать даты занятий, сессий, культурных и спортивных событий, устанавливать напоминания и не пропускать важные мероприятия. Оно будет полезно для повышения организованности учебного процесса в колледже.
2. Цели и задачи
Цель: создать удобное мобильное приложение-календарь для планирования событий колледжа с возможностью добавления, редактирования, удаления событий и получения уведомлений о них.
Задачи:
Реализовать интерфейс календаря с отображением месяцев и дней.
Позволить пользователям добавлять события на конкретную дату (название, время, место, тип события).
Обеспечить возможность просмотра, редактирования и удаления событий.
Хранить все события в локальной базе данных.
Реализовать систему напоминаний (уведомления за 15 минут до события).
3. Функциональные требования
3.1. Основные функции приложения
Просмотр календаря:
Отображение текущего месяца с сеткой дней.
Переключение между месяцами (вперед/назад).
Визуальное выделение дней, на которые запланированы события (точка или цвет фона).
Управление событиями:
При нажатии на день открывается список событий на эту дату.
Добавление нового события с указанием:
Названия (обязательно)
Времени начала (и опционально – окончания)
Места проведения
Типа события (Лекция, Семинар, Спорт, Культура, Другое)
Редактирование существующего события.
Удаление события с подтверждением.
Хранение данных:
Все события сохраняются в локальной базе данных SQLite (Room).
После перезапуска приложения данные сохраняются.
Напоминания:
Автоматическая установка напоминания за 15 минут до события.
Возможность включения/отключения напоминаний в настройках.
При нажатии на уведомление открывается экран с событиями на нужную дату.
Настройки (опционально):
Изменение времени уведомления (10, 15, 30 минут).
Включение/отключение всех напоминаний.
3.2. Нефункциональные требования
Приложение должно иметь простой и понятный интерфейс (Material Design).
Приложение должно быть доступно на устройствах с Android версии не ниже 7.0 (API 24).
Все данные должны храниться в локальной базе данных (использование Room / SQLite).
Приложение должно работать без интернета (полностью офлайн).

5. Архитектура приложения
Модель (Entity): Классы Event, возможно UserSettings.
DAO (Data Access Object): Интерфейсы для CRUD операций над событиями (вставка, чтение по дате, обновление, удаление).
Repository: Единый источник данных (работа с БД).
ViewModel (опционально): Хранение данных событий при повороте экрана.
Activity/Fragment:
MainActivity – отображение календаря.
EventListActivity – список событий на выбранную дату.
EventFormActivity – добавление/редактирование события.
SettingsActivity – настройки напоминаний.
6. Этапы разработки
Анализ требований и проектирование интерфейса (макеты экранов календаря, списка событий, формы).
Реализация базы данных Room (Entity, DAO).
Разработка основного экрана календаря (адаптер для сетки месяцев).
Реализация CRUD операций (добавление, редактирование, удаление событий).
Реализация системы напоминаний (AlarmManager + BroadcastReceiver + уведомления).
Тестирование приложения на эмуляторе и реальном устройстве (если есть).
Подготовка документации и отчета по курсовой работе.
7. Ожидаемые результаты
На выходе должно получиться полностью функциональное Android-приложение «Календарь событий колледжа», позволяющее:
просматривать календарь по месяцам;
добавлять, редактировать и удалять события;
получать уведомления о предстоящих событиях;
хранить все данные локально без потери после перезапуска.
8. Заключение
Данное техническое задание описывает основные цели, задачи и функциональные требования к разработке приложения «Календарь событий колледжа». Реализация этого проекта позволит закрепить навыки разработки на Java под Android, работы с базами данных Room, адаптерами, уведомлениями и жизненным циклом Activity.
